# Implementation Plans

Plans translate accepted architecture and product requirements into verifiable delivery work. Use a
plan for multi-step changes, migrations, cross-cutting work, or changes with operational risk.

## Conventions

- Name plans `YYYY-MM-DD-topic.md` using a lowercase hyphenated topic.
- Copy [the template](0000-template.md); do not modify the template for a single plan.
- Link relevant architecture sections and ADRs. A plan may implement a decision but must not silently
  replace one.
- Make scope, acceptance criteria, exact paths, commands, operational effects, and rollback explicit.
- Keep completed plans as delivery history; record material architectural outcomes in ADRs.
